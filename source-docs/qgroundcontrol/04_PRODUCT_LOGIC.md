# Продуктовая логика QGroundControl

Документ описывает QGroundControl как продукт: его функции, пользовательские сценарии, роли, сущности и бизнес-правила. Фокус — на том, **что система делает для пользователя**, а не на том, как она устроена изнутри.

---

## 1. Основные функции системы

QGroundControl — это единая наземная станция управления (GCS), которая выполняет **четыре ключевых продуктовых функции**:

### 1.1 Управление полётом / движением (Fly)
Основной экран работы оператора. Позволяет в реальном времени наблюдать за аппаратом на карте, видеть телеметрию (скорость, высота, GPS, батарея, heading) и отдавать команды: arming, takeoff, pause, return, go-to-location, change speed, emergency stop.

### 1.2 Планирование миссий (Plan)
Визуальный редактор маршрутов. Пользователь расставляет путевые точки (Waypoints) на карте, настраивает поведение в каждой точке (съёмка, разворот, задержка), задаёт геозоны (GeoFence) и точки аварийного возврата (Rally Points), после чего загружает план на борт аппарата.

### 1.3 Настройка и калибровка аппарата (Configure)
Экран конфигурации позволяет: обновлять прошивку автопилота, калибровать датчики (компас, акселерометр, гироскоп), настраивать тип рамы (frame), регулировать PID-параметры, привязывать каналы пульта управления (RC), настраивать режимы полёта.

### 1.4 Анализ и диагностика (Analyze)
Набор инженерных инструментов: просмотр бортовых логов (Onboard Logs), интерактивная MAVLink-консоль, инспектор MAVLink-сообщений, визуализация вибраций, гео-теггинг фотографий.

### 1.5 Настройки приложения (Settings)
Настройки самого интерфейса QGC: единицы измерения, видеопоток, Autoconnect, выбор тайлов карт (Online/Offline), конфигурация связи, виртуальный джойстик, RemoteID.

## 2. Ключевые пользовательские сценарии

### Сценарий 1: Первый запуск и настройка нового аппарата

```
User → Подключает аппарат по USB
  System → Обнаруживает устройство, запускает синхронизацию параметров
  System → Отображает прогресс-бар загрузки параметров
User → Открывает Configure
User → Выбирает "Firmware" → загружает/обновляет прошивку
  System → Скачивает файл прошивки, прошивает контроллер, перезагружает его
User → Выбирает "Frame" → указывает тип рамы (квадрокоптер / лодка / ровер)
  System → Записывает FRAME_CLASS параметр на борт
User → Выбирает "Sensors" → запускает калибровку компаса
  System → Показывает инструкцию: поворачивайте аппарат
  System → Получает данные магнитометра, считает смещения, записывает на борт
User → Выбирает "Radio" → калибрует пульт
  System → Просит двигать стики в крайние положения
  System → Запоминает диапазоны каналов RC, записывает параметры
```

### Сценарий 2: Планирование и выполнение автономной миссии

```
User → Открывает Plan View
User → Кликает на карту → создаёт Waypoints
  System → Добавляет точку в список, рисует линию маршрута на карте
  System → Рассчитывает статистику: расстояние, время полёта, расход батареи
User → Настраивает параметры точки (высота, скорость, действие в точке)
User → Рисует GeoFence (область допустимого полёта)
  System → Формирует геозону, визуализирует на карте
User → Нажимает "Upload" (стрелка вверх)
  System → Передаёт миссию на борт по протоколу MAVLink (MISSION_COUNT → MISSION_ITEM)
  System → Получает MISSION_ACK → показывает "Upload complete"
User → Переключается в Fly View
User → Нажимает Arm → подтверждает (Delay Button)
  System → Отправляет MAV_CMD_COMPONENT_ARM_DISARM
User → Нажимает Start Mission → подтверждает
  System → Переводит аппарат в Mission Mode
  System → Аппарат начинает движение по точкам
  System → На карте подсвечиваются пройденные Waypoints, рисуется трек
```

### Сценарий 3: Ручное управление в движении

```
User → Наблюдает за аппаратом на карте в Fly View
User → Кликает на карту → выбирает "Go to location"
  System → Показывает маркер "Go here" на карте
  System → Запрашивает подтверждение (Delay Button)
User → Подтверждает удержанием кнопки
  System → Отправляет guidedModeGotoLocation() на борт
  System → Аппарат меняет курс к указанной точке
User → Нажимает Pause
  System → Аппарат останавливается / зависает
User → Изменяет скорость (Change Speed → ползунок)
  System → Отправляет MAV_CMD_DO_CHANGE_SPEED
User → Нажимает RTL (Return)
  System → Аппарат разворачивается и движется к точке запуска
```

### Сценарий 4: Потеря связи (Failsafe)

```
System → Перестаёт получать Heartbeat от аппарата (timeout > N секунд)
  System → Показывает "Communication Lost" в toolbar
  System → Появляется кнопка "Disconnect"
  System → Все Guided-команды блокируются
  → Аппарат самостоятельно выполняет Failsafe (RTL / Land / Hold — в зависимости от настроек на борту)
System → Связь восстановлена → heartbeat снова приходит
  System → Убирает предупреждение, разблокирует UI
```

### Сценарий 5: Завершение миссии

```
System → Аппарат достигает последнего Waypoint
System → В зависимости от настроек: зависает / садится / выполняет RTL
System → Аппарат disarm-ится (моторы выключены)
  System → Открывается диалог "Flight Plan complete"
User → Выбирает "Remove plan from vehicle" или "Leave plan on vehicle"
  System → Очищает миссию на борту (или оставляет)
User → (опционально) Нажимает "Resume Mission from Waypoint N"
  System → Перестраивает миссию с указанной точки для повторного запуска
```

### Сценарий 6: Работа с несколькими аппаратами

```
System → Обнаруживает Heartbeat от второго аппарата
  System → Создаёт второй экземпляр Vehicle
  System → Показывает Multi-Vehicle Panel (верхний правый угол Fly View)
  System → Обе иконки отображаются на карте
User → Кликает на аппарат в списке → он становится активным
  System → Телеметрия переключается на выбранный аппарат
User → Выделяет несколько аппаратов (чекбоксы)
User → Нажимает "Start" (массовый запуск миссий)
  System → Отправляет startMission() каждому выбранному
```

## 3. Роли пользователей

В QGC **нет формальной системы аутентификации и ролей** (RBAC). Однако продуктовая логика подразумевает функциональное разделение по экранам и уровню доступа:

| Функциональная роль | Экран | Что делает | Ограничения |
|---|---|---|---|
| **Пилот / Оператор** | Fly View | Наблюдает за полётом, отдаёт Guided-команды (Arm, RTL, Pause, GoTo), управляет джойстиком | Все действия требуют подтверждения (Delay Button). Emergency Stop требует удержания. |
| **Планировщик миссий** | Plan View | Составляет маршрут, расставляет Waypoints, GeoFence, Rally Points. Загружает/выгружает план | Не может управлять аппаратом в реальном времени из Plan View |
| **Инженер / техник** | Configure | Обновляет прошивку, калибрует датчики, настраивает PID и параметры автопилота | При калибровке навигация между экранами блокируется (`navigationBlockedReason`) |
| **Аналитик / отладчик** | Analyze | Анализирует логи, инспектирует MAVLink, проверяет вибрации | Экран доступен только в Advanced Mode |
| **Администратор QGC** | Settings | Настраивает соединения, единицы, карты, RemoteID, видеопоток | — |

**Advanced Mode** — скрытый режим, активируемый через Shift+Click (или long press на мобильных) по номеру версии. Открывает доступ к экрану Analyze и расширенным параметрам.

## 4. Основные сущности системы

| Сущность | Описание | Жизненный цикл |
|---|---|---|
| **Vehicle** | Цифровой двойник подключённого аппарата. Содержит координаты, статус arming, режим полёта, батарею, GPS, trajectory. | Создаётся при Heartbeat → живёт пока есть связь → удаляется при Disconnect |
| **Fact** | Единица данных (параметр или телеметрия). Имеет значение, тип, min/max, единицы измерения, default. | Загружается при подключении → обновляется MAVLink → синхронизируется с UI |
| **Mission (Plan)** | Набор путевых точек + GeoFence + Rally Points. Формат хранения: `.plan` файл (JSON). | Создаётся в Plan View → загружается на борт (Upload) → выполняется → завершается |
| **Waypoint** | Одна точка маршрута с координатами, высотой, действием и параметрами. | Создаётся кликом на карту → редактируется → отправляется на борт в составе Mission |
| **GeoFence** | Геометрическая зона (полигон или круг), определяющая допустимую область движения. | Создаётся в Plan View → загружается на борт → автопилот применяет Failsafe при выходе за границу |
| **Rally Point** | Альтернативная точка возврата (вместо Home). | Создаётся в Plan View → загружается на борт → используется при RTL как ближайшая |
| **Link** | Канал связи (Serial, UDP, TCP) между QGC и аппаратом. | Создаётся в Settings или авто-обнаруживается → живёт пока открыт порт → закрывается при Disconnect |
| **Firmware Plugin** | Абстракция прошивки (PX4 / ArduPilot). Определяет доступные режимы полёта, команды, поведение. | Инстанцируется автоматически по типу автопилота из Heartbeat |
| **Video Stream** | Поток видео с борта (RTSP/UDP через GStreamer). | Запускается при наличии URI → отображается в PiP или full screen → останавливается при Disconnect |

## 5. Как пользователь взаимодействует с системой

### Навигация между экранами

Пользователь переключается между экранами через глобальное **меню-гамбургер** (клик по лого QGC в toolbar). Доступные пункты:

| Кнопка | Экран | Всегда видна |
|---|---|---|
| **Fly** | Fly View — экран полёта / движения | Да |
| **Plan** | Plan View — планировщик миссий | Да |
| **Analyze** | Analyze — логи, MAVLink, вибрации | Только в Advanced Mode |
| **Configure** | VehicleConfigView — настройка аппарата | Да |
| **Settings** | AppSettings — настройки приложения | Да (если не объединён с Configure) |
| **Close** | Завершение приложения | Да |

### Паттерн взаимодействия

```
┌─────────────────────────────────────────────────────┐
│                      TOOLBAR                         │
│  [Logo/Menu] [Status] [FlightMode] ... [Indicators] │
├──────────────────────┬──────────────────────────────┤
│                      │                              │
│   Основная рабочая   │   Контекстная правая панель  │
│   область (карта /   │   (телеметрия / параметры /  │
│   видео / форма)     │    список компонентов)       │
│                      │                              │
└──────────────────────┴──────────────────────────────┘
```

- **Toolbar** — постоянный контекст: статус Vehicle, режим полёта, индикаторы здоровья.
- **Основная область** — карта (Fly/Plan) или форма конфигурации (Configure/Settings).
- **Правая панель** — контекстная: инструменты миссии (Plan), телеметрия (Fly), список компонентов (Configure).

### Модальность и подтверждения

Система использует **три уровня модальности** для защиты от случайных действий:

1. **Delay Button (GuidedActionConfirm)** — для критических команд (Arm, Emergency Stop, Start Mission). Требует **удержания кнопки** ~2 секунды.
2. **Value Slider (GuidedValueSlider)** — для команд с параметром (Takeoff: высота, Change Speed: скорость). Пользователь устанавливает значение ползунком, затем подтверждает Delay Button.
3. **Dialog (MessageDialog)** — для необратимых действий (удалить прошивку, очистить план, закрыть с несохранёнными данными).

## 6. Ограничения и бизнес-правила

### Правила arming и запуска

| Правило | Формулировка | Следствие |
|---|---|---|
| **Checklist Gate** | Если включен `enforceChecklist`, arm/takeoff/start mission заблокированы до прохождения Pre-Flight Checklist | Кнопки Arm и Start Mission неактивны, пока `checkListState ≠ CheckListPassed` |
| **Health Check Gate** | Если автопилот поддерживает Health & Arming Check Report, система проверяет `canArm`, `canTakeoff`, `canStartMission` | Кнопки заблокированы до устранения ошибок на борту |
| **RC RSSI Gate** | Опционально: guided-команды заблокированы при отсутствии RC сигнала (`guidedActionsRequireRCRSSI`) | Без пульта нельзя управлять аппаратом |

### Правила навигации между экранами

| Правило | Формулировка |
|---|---|
| **Validation Error Lock** | Нельзя переключить экран, если есть поля параметров с невалидными значениями (`validationErrorCount > 0`) |
| **Calibration Lock** | Нельзя покинуть Configure во время калибровки (`navigationBlockedReason ≠ ""`) |
| **Unsaved Mission Warning** | При закрытии приложения с несохранённой миссией — предупреждение с вариантами Yes/No |
| **Pending Parameter Writes Warning** | При закрытии с незаписанными параметрами — предупреждение |
| **Active Connection Warning** | При закрытии с активным подключением — подтверждение выхода |

### Правила отображения действий

| Правило | Формулировка |
|---|---|
| **No Vehicle = No Actions** | Если нет подключённого аппарата → ToolStrip не показывает кнопки, Guided Actions недоступны |
| **Flying = Different Actions** | Набор доступных кнопок полностью зависит от состояния: на земле (Arm, Start Mission) vs в движении (Pause, RTL, Land, GoTo) |
| **Mission Active = Locked** | Во время активной миссии нельзя вручную менять высоту или орбитировать (чтобы не конфликтовать с автопилотом) |
| **Emergency Stop Visibility** | Кнопка Emergency Stop видна ТОЛЬКО когда аппарат заармлен И летит одновременно |

### Правила связи

| Правило | Формулировка |
|---|---|
| **Heartbeat Timeout** | Если в течение N секунд не получен HEARTBEAT → состояние **Communication Lost** |
| **Communication Lost → Actions Locked** | В этом состоянии Guided-действия заблокированы, появляется кнопка Disconnect |
| **High Latency Link** | При подключении через High Latency канал (спутник) → Virtual Joystick автоматически скрывается (слишком высокий Latency для ручного управления) |

### Правила данных

| Правило | Формулировка |
|---|---|
| **Fact Validation** | Каждый параметр (`Fact`) имеет `min` и `max`. Значения за пределами автоматически отклоняются |
| **Offline Editing** | Plan View позволяет создавать миссии без подключённого аппарата (Offline Editing Vehicle) |
| **Terrain Skip for Boats/Subs** | Для аппаратов типа `MAV_TYPE_SURFACE_BOAT` и `MAV_TYPE_SUBMARINE` рельеф местности не загружается |
| **Vehicle-specific Checklist** | Pre-Flight Checklist автоматически выбирается по типу аппарата: MultiRotor / FixedWing / VTOL / Rover / Sub |

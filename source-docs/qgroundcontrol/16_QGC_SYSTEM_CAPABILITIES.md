## 1. EXECUTIVE SUMMARY

**QGroundControl (QGC)** — наземная станция управления для беспилотных аппаратов любого типа: планирование, полёт, настройка, анализ — в одном приложении.

| | |
|---|---|
| **Позиционирование** | Единственная open-source GCS, полностью поддерживающая PX4 и ArduPilot на одной платформе |
| **Ключевая ценность** | Заменяет 3–5 инструментов одним: прошивка → настройка → миссия → полёт → анализ |
| **Типы аппаратов** | Мультикоптеры, самолёты, VTOL, роверы, лодки, подводные аппараты, дирижабли |
| **Платформы** | Windows, macOS, Linux, Android, iOS — единый codebase |
| **Open Source** | $0 лицензии + Plugin API для создания кастомных branded-версий |
| **Regulatory readiness** | Remote ID, GeoFence, ADSB — готовность к FAA / EASA без доработок |
| **Применение** | Агросъёмка, инспекция инфраструктуры, морские операции, SAR, коммерческие флоты |

## 2. FUNCTIONAL DOMAINS

8 доменов, из которых 6 — критичны для эксплуатации:

```
┌─────────────────────────────────────────────────────────────┐
│                    QGroundControl                           │
├──────────────────┬──────────────────┬───────────────────────┤
│ 1. Connection    │ 2. Mission       │ 3. Flight             │
│    [HIGH]        │    Planning      │    Operations          │
│                  │    [HIGH]        │    [HIGH]              │
├──────────────────┼──────────────────┼───────────────────────┤
│ 4. Telemetry     │ 5. Vehicle       │ 6. Safety             │
│    [HIGH]        │    Config [HIGH] │    [HIGH]              │
├──────────────────┼──────────────────┼───────────────────────┤
│ 7. Camera &      │ 8. Analysis      │                       │
│    Payload [MED] │    & Maps [MED]  │                       │
└──────────────────┴──────────────────┴───────────────────────┘
```

> **HIGH** — без этого система непригодна для эксплуатации.
> **MED** — расширяет ценность, но базовые операции возможны без него.

---

### Domain 1 — Vehicle Connection ⬤ HIGH

> **Суть:** Подключение GCS к аппарату. 5 типов каналов + автообнаружение.
> **Решает:** Надёжная связь в любых условиях — от USB на столе до WiFi в поле.

| Компонент | Что это даёт |
|-----------|-------------|
| **Serial / UDP / TCP** | Plug-and-play + работа по сети и на дистанции |
| **Bluetooth** | Управление с мобильного устройства в поле |
| **Auto-Connect** | Нулевая ручная настройка — подключил и работаешь |
| **Multi-Vehicle** | Управление несколькими аппаратами из одного интерфейса |
| **MAVLink 2 Signing** | Защита от перехвата и подмены команд |
| **Log Replay** | Разбор инцидентов без реального ЛА |

**→ Любой оператор подключается за минуты. 5 типов каналов покрывают все сценарии.**

---

### Domain 2 — Mission Planning ⬤ HIGH

> **Суть:** Визуальный редактор миссий на карте. Планирование сложных задач без программирования.
> **Решает:** Survey, облёт объектов, учёт рельефа — всё drag-and-drop.

| Возможность | Что это даёт оператору |
|-------------|----------------------|
| **Waypoints** | Маршрут за 2 минуты |
| **Survey Pattern** | Покрытие территории для ортофотоплана — одной кнопкой |
| **Corridor Scan** | Инспекция линейных объектов (дороги, ЛЭП, трубопроводы) |
| **Structure Scan** | Облёт 3D-объектов для полной инспекции |
| **Landing Patterns** | Безопасная посадка самолётов/VTOL без ручного расчёта |
| **GeoFence + Rally** | Зоны ограничений + запасные площадки |
| **Camera Calc + Terrain** | Гарантия качества съёмки на любом рельефе |
| **Mission Statistics** | Оценка расхода батареи и времени — до старта |

**→ Главная продуктовая функция QGC. Survey + Corridor + Structure Scan на одной карте — уровень коммерческих GCS за $5–15K. Здесь — бесплатно.**

---

### Domain 3 — Flight Operations ⬤ HIGH

> **Суть:** Полный набор команд для управления в реальном времени.
> **Решает:** Оператор сохраняет полный контроль — от взлёта до экстренной остановки.

| Категория | Команды | Приоритет |
|-----------|---------|-----------|
| **Базовый цикл** | Arm, Takeoff, Land, RTL | HIGH |
| **Оперативное управление** | GoTo, Orbit, ROI, Change Alt/Speed/Heading | HIGH |
| **Экстренные** | Pause, Emergency Stop, Abort Landing | HIGH |
| **Миссии** | Start / Continue Mission, Flight Mode Switch | HIGH |
| **Ручное управление** | Joystick, Virtual Joystick | MEDIUM |
| **Доп. оборудование** | Follow Me, Gripper, Landing Gear | LOW |

**→ 20+ команд реального времени. Оператор — не наблюдатель, а командир. Это требование регуляторов для BVLOS-операций.**

---

### Domain 4 — Telemetry & Monitoring ⬤ HIGH

> **Суть:** «Приборная панель» — 25+ групп данных с аппарата в реальном времени.
> **Решает:** Оператор должен знать, что происходит с ЛА — прямо сейчас.

| Приоритет | Группы данных |
|-----------|--------------|
| **HIGH** | GPS, Battery, Attitude, Altitude, Speed, Link Quality, Health Check |
| **MEDIUM** | Wind, Vibration, ESC/RPM/Temperature, Distance Sensors |
| **LOW** | EFI, Generator, Hygrometer (промышленные сенсоры) |

**→ Телеметрия = основа для решений. Увидеть, что батарея садится на 20% быстрее из-за ветра, и вернуть аппарат — это спасение оборудования стоимостью $5–50K.**

---

### Domain 5 — Vehicle Setup & Configuration ⬤ HIGH

> **Суть:** Полный мастер конфигурации — от прошивки до PID.
> **Решает:** Настройка дрона обычно требует 3–5 инструментов. QGC делает всё в одном GUI.

| Приоритет | Компоненты |
|-----------|-----------|
| **HIGH** | Firmware Upgrade, Airframe, Sensors, Radio, Flight Modes, Power, Safety/Failsafes |
| **MEDIUM** | PID Tuning, Autotune, Motor Test, Parameter Editor (1000+ параметров) |
| **LOW** | Joystick Setup, Lights, Sub Frame, Heli (специфичные типы ЛА) |

**→ Новый контроллер из коробки → прошивка → калибровка → первый полёт — не закрывая QGC. Один интерфейс для PX4 и ArduPilot.**

---

### Domain 6 — Safety & Compliance ⬤ HIGH

> **Суть:** Функции безопасности и регуляторного соответствия.
> **Решает:** Без GeoFence и Remote ID коммерческие полёты незаконны.

| Приоритет | Возможность | Зачем |
|-----------|------------|-------|
| **HIGH** | GeoFence | Предотвращает вылет за разрешённую зону |
| **HIGH** | Remote ID (ODID) | Обязателен для коммерческих полётов US/EU (2024+) |
| **HIGH** | Pre-flight Checks | Автоматическая проверка готовности перед взлётом |
| **MEDIUM** | Object Avoidance, ADSB | Предотвращение столкновений |
| **MEDIUM** | MAVLink Signing | Защита канала управления |

**→ Безопасность — не feature, а лицензия на работу. QGC покрывает все требования FAA и EASA из коробки.**

---

### Domain 7 — Video, Camera & Payload ⬤ MEDIUM

> **Суть:** Видео, камера, гимбал, термал — управление съёмкой из GCS.
> **Решает:** Для инспекции и картографии камера так же важна, как сам полёт.

| Приоритет | Возможности |
|-----------|------------|
| **HIGH** | Video Streaming (RTSP/GStreamer), Gimbal Control, Photo/Video Capture |
| **MEDIUM** | Camera Settings, Thermal, Object Tracking, Zoom, On-screen Gimbal |
| **LOW** | Subtitle Writer, GeoTag |

**→ Без камеры дрон — «летающий GPS-трекер». Полная интеграция видео и гимбала превращает QGC в рабочую станцию оператора.**

---

### Domain 8 — Analysis, Logs & Maps ⬤ MEDIUM

> **Суть:** Пост-полётный анализ, логи, работа с картами оффлайн.
> **Решает:** Понять, что произошло после полёта. Работать без интернета.

| Приоритет | Возможности |
|-----------|------------|
| **HIGH** | Offline Maps (80% операций без интернета), Telemetry Log, Log Download |
| **MEDIUM** | Log Replay, MAVLink Inspector, Vibration Analysis, GeoTag, 7+ Map Providers |
| **LOW** | 3D Viewer, MAVLink Console |

**→ Автозапись телеметрии + оффлайн-карты + диагностика = профессиональные операции без зависимости от интернета.**

## 4. KEY PRODUCT STRENGTHS

### 🏆 1. Два автопилота — один интерфейс

PX4 и ArduPilot через отдельные плагины с собственными режимами, калибровкой, PID. Ни одна другая GCS не поддерживает оба стека на одном уровне. Для организаций с разнородным парком — единственный выбор.

### 🏆 2. Коммерческое планирование миссий — бесплатно

Survey, Corridor Scan, Structure Scan, Terrain Following, Camera Calc. В коммерческих GCS — $5–15K. В QGC — $0. Порог входа для сложных автономных миссий = ноль.

### 🏆 3. Полный lifecycle в одном окне

Прошивка → Калибровка → Миссия → Полёт → Анализ. Нет переключения между инструментами. Меньше ошибок, быстрее подготовка.

### 🏆 4. Regulatory-ready

Remote ID, GeoFence, ADSB, MAVLink Signing, Pre-flight Checklists. Всё встроено. Легальные коммерческие полёты — без дополнительного ПО.

### 🏆 5. Платформа, а не приложение

Plugin API позволяет OEM-партнёрам создавать branded GCS: свой UI, карты, логика. Без поддержки полного форка. Пример: Auterion Mission Control.

### 🏆 6. Один инструмент — 7+ типов ЛА

Мультикоптеры, самолёты, VTOL, роверы, лодки, Sub, дирижабли. Архитектура `FirmwarePlugin` + `MAV_TYPE` адаптирует интерфейс автоматически. Одна команда операторов — весь парк.

## 5. KEY SYSTEM INSIGHTS

### 🔍 1. Монолит, но с чёткими границами

Весь код — один процесс. Внутри — модули с отдельными CMakeLists.

**→ Продукт:** Работает быстро и стабильно.
**→ Модернизация:** Выделение API/backend потребует рефакторинга. Модули связаны сильнее, чем кажется.

### 🔍 2. Vehicle = центр всего

Один класс (~189KB) отвечает за состояние, MAVLink, телеметрию, команды, 20+ групп данных.

**→ Продукт:** Удобно — всё доступно через `vehicle.*`.
**→ Модернизация:** Первый кандидат на декомпозицию. Без разделения — масштабирование невозможно.

### 🔍 3. Fact System — ядро, которое нельзя потерять

1000+ параметров PX4/ArduPilot → единая абстракция с типизацией, валидацией и привязкой к UI.

**→ Продукт:** Пользователь не замечает разницы между прошивками.
**→ Модернизация:** Сохранить Fact System обязательно. Без неё dual-firmware support разрушится.

### 🔍 4. FirmwarePlugin = масштабируемость

Вся firmware-specific логика изолирована. Core не знает, какая прошивка.

**→ Эффект:** Новый автопилот = новый плагин. Без правок в ядре. Паттерн переносится в любую архитектуру.

### 🔍 5. Готовность к API-first трансформации

Бизнес-логика уже организована через `Q_PROPERTY` (данные) и `Q_INVOKABLE` (команды) — де-факто API-контракт.

**→ Эффект:** Разделение на Backend + Frontend — это рефакторинг существующих интерфейсов, а не переписывание с нуля.

## 6. POSITION IN MARKET

| Аспект | Оценка |
|--------|--------|
| **Где сильнее всего** | Dual-firmware support (PX4 + ArduPilot) — ни одна другая GCS не покрывает оба стека. Планирование миссий (Survey, Structure Scan) — коммерческий уровень, $0 стоимости. |
| **Где конкурентоспособна** | Кроссплатформенность (5 OS), regulatory compliance (Remote ID, GeoFence, ADSB), Plugin API для OEM. |
| **Где ограничена** | Монолитная desktop-архитектура. Нет cloud/web-версии. Нет нативного API для интеграции с внешними системами. UI зависит от Qt/QML — ограниченные возможности для кастомизации без знания фреймворка. |
| **Максимальная ценность** | Организации с гетерогенным парком ЛА (PX4 + ArduPilot), полевые операции без интернета, коммерческие операторы, которым нужен regulatory-ready инструмент без лицензионных затрат. |

## 7. FINAL TAKEAWAY

**QGroundControl — это не просто приложение. Это платформа.**

| Уровень | Что это значит |
|---------|---------------|
| **Как инструмент** | Полный цикл работы с ЛА: прошивка → настройка → миссия → полёт → анализ. В одном окне, на любой платформе. |
| **Как продукт** | Замена коммерческих GCS стоимостью $5–15K. Open-source, но enterprise-уровень функциональности. Regulatory-ready. |
| **Как платформа** | Plugin API превращает QGC в основу для branded GCS. OEM-партнёры строят свои продукты поверх — без полного форка. |
| **Как стратегический актив** | ~95% совместимость с рынком open-source дронов. 7+ типов ЛА. 5 платформ. Fact System + FirmwarePlugin — архитектура, готовая к трансформации в API-first. |

**Где QGC сильнее всего:**

> Единственная точка, где **два автопилота** (PX4 + ArduPilot) встречаются в **одном интерфейсе** с **коммерческим уровнем планирования миссий** и **regulatory compliance из коробки** — при **нулевой стоимости лицензии**.
